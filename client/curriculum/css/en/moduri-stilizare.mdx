  The inline method comes with <strong>huge disadvantages</strong>, which is why most programmers avoid it. Let's see why it's not recommended:

  {<List variant="bullets">
    <li className="mb-4">if we have to style multiple similar elements (let's say multiple paragraphs that we want to look the same), with the inline method we will have to copy-paste the code for each paragraph separately. Moreover, when we change our mind about the styles, we will have to remember to modify them everywhere.
    </li>

  If the inline method requires us to style each element individually, with an <strong>internal CSS</strong> we can control the style for an entire page, thus avoiding duplicate code. Also, we will have a better understanding of the code and save time.

  The method of integrating CSS code into a web page is done by inserting the <FormattedText as="strong">{'<style>'}</FormattedText> element in the <FormattedText as="strong">{'<head>'}</FormattedText> section of the HTML document. Compared to the previous method, we must specify for which elements we will apply the styling rules.

  <Highlight className="my-5" language="html" code={`<style>
  a {
    color: #000;
    text-decoration: none;
    border-bottom: 1px solid #000;
  }
</style>`} />

  In the example above, we apply the rules to all <FormattedText as="strong">a</FormattedText> elements, using the tag selector of the HTML element.

  {<LessonTip> There are also other selectors: by class or id. We will talk about them in the following lessons, but for now, it is important to remember this styling method and the role of selectors. </LessonTip>}

  In practice, however, <strong>we do not recommend using the internal method</strong> because websites are built on multiple pages, so we will end up copy-pasting on each page. Plus, although the HTML and CSS code is slightly separated, they are still in the same file, and we will end up with <span className="text-bold">spaghetti code</span> as the application grows.

  And we end up where we started, right?

  Well, having said all that, what is the solution to do things right? The <strong>external CSS method</strong> all the way!

  This is <strong>the method we recommend</strong> and it consists of creating a separate file that we associate with the HTML pages. The CSS file is integrated into HTML using the <FormattedText as="strong">{'<link>'}</FormattedText> element.

  <Highlight className="my-5" language="html" code={`<link rel="stylesheet" href="./style.css">`} />
 
  Where:

  {<List variant="bullets">
    <li className="mb-4">The <FormattedText as="strong">rel</FormattedText> attribute specifies the relationship between the current HTML document and the external document we are connecting to it. In our case, the external document is a style sheet.</li>
    <li>The <FormattedText as="strong">href</FormattedText> attribute specifies the location (URL) where we find the style sheet.</li>
  </List>}

  More specifically, <span className="text-bold">let's see an example</span>:

   With this method, we solve both problems:

  {<List variant="bullets">
    <li className="mb-4">In terms of copy-pasting, having a file included in multiple pages is enough to modify the style in that file, and the styles will be applied wherever it is included.</li>
    <li>Regarding spaghetti code, having the styles separated from HTML makes it much easier to understand and modify the code.</li>
  </List>}
