  Unlike other languages, we can write invalid HTML and our site will still work, most of the time exactly the way we want it to. The reason is the following:

  <LessonQuote variant="centered"> Instead of showing a blank page, or an error message, the browser tries to solve the problems in order to show something to the users. </LessonQuote>

  In other words, let's assume we have the following code snippet where we didn't correctly close the <FormattedText as="span">{'</h1>'} </FormattedText> tag:

  <Highlight language="html" className="my-5" code={`<h1> FrontEnd.ro </h2>
<p>
  Bine ai venit și spor la 
  învățat.
</p>`} />

  However, if we open the browser we'll see the result we wanted: a title followed by a text paragraph. The browser notices our error and tries to fix it by replacing <FormattedText as="span"> {'</h2>'} </FormattedText> with <FormattedText as="span"> {'</h1>'} </FormattedText>.
